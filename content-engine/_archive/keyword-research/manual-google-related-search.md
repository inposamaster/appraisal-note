# Google 연관검색어 — 수동 확인 절차

## 목적

검색 의도를 확장하고, **검색결과 1페이지 경쟁**을 관찰해 글 작성 가능성을 판단한다.

## 사전 준비

- 템플릿: `raw-data/google-related-search/_template.md`
- 확인할 키워드: `keywords.csv`의 `keyword` 또는 자동완성에서 나온 후보

## 절차

### 1. Google에서 키워드 검색

- 시크릿 모드 권장
- 검색어를 정확히 입력 (자동완성에서 고른 표현 그대로)

### 2. 관련 검색어 기록

다음 위치에서 관련 표현을 수집한다.

- 검색 결과 **하단** 관련 검색어
- 검색 결과 **중간·상단** 관련 검색어 (노출 시)

| 기록 항목 | 내용 |
|---|---|
| 관련 검색어 | 표현 그대로 |
| 글 작성 가능성 | 높음 / 중간 / 낮음 |
| 메모 | 왜 이 후보가 유망한지 |

### 3. 1페이지 경쟁자 유형 관찰

상위 10개 내외를 스캔하고 **유형**을 분류한다.

| 유형 | 예시 |
|---|---|
| 정부·공공 | 국세청, 법원, 정부24 |
| 언론·대형 매체 | 신문, 포털 뉴스 |
| 대형 플랫폼 | 네이버 블로그 상위, 티스토리 상위, 유튜브 |
| 전문 사무소·업체 | 감정평가법인, 세무사 |
| 개인 블로그 | 소규모 블로그 |
| Q&A | 지식iN, Quora, Reddit |

### 4. 판단 기준

| 신호 | 해석 |
|---|---|
| 대형 사이트만 1페이지 | 후순위·보류 검토 |
| 개인 블로그·Q&A 혼재 | 신생 블로그 진입 가능성 있음 |
| 직접 답변 부족 | 콘텐츠 기회 |
| 법률·세금 단정 콘텐츠 다수 | 사실 확인·면책 강화 필요 |

### 5. 글 작성 가능성·보류 사유

- **높음**: 진입 여지 + 의도 명확
- **중간**: 경쟁 있으나 차별화 가능
- **낮음**: 대형 사이트 독점
- **보류**: 리스크·의도 불명·중복 주제

`competition_note`에 "1페이지: ○○ 유형 다수" 형태로 요약한다.

### 6. 파일 저장

- 경로: `raw-data/google-related-search/YYYY-MM-DD.md`

### 7. keywords.csv 반영

- `competition_note` 업데이트
- `google_related_found` (확장 열 적용 시): Y / N
- `validation_status`: 검증중 → 검증완료 (다른 소스 확인 후)
