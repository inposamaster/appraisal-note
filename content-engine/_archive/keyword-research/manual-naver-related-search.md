# Naver 자동완성·연관검색어 — 수동 확인 절차

## 목적

한국어 검색 의도를 확인하고, **네이버 VIEW·블로그** 노출 구조를 관찰한다. 구글과 네이버 전략을 구분한다.

## 사전 준비

- 템플릿: `raw-data/naver-related-search/_template.md`
- 동일 seed/keyword를 Google 절차와 **교차 비교**할 것

## 절차

### 1. 네이버 검색창 자동완성

- seed keyword 입력 시 자동완성 목록 기록
- Google 자동완성과 **다른 표현**에 주목 (한국어 어미, 네이버 사용자 패턴)

### 2. 연관검색어 확인

- 검색 결과 상단·하단 연관검색어 기록
- Google에 없는 네이버 특화 표현 메모

### 3. VIEW·블로그 영역 관찰

| 확인 항목 | 기록 |
|---|---|
| VIEW 탭에 개인 블로그 노출 | 예 / 아니오 |
| 블로그 탭 상위 콘텐츠 유형 | 공인중개, 세무사, 개인 등 |
| 지식iN·카페 혼재 | 예 / 아니오 |

**지식iN, 카페, 블로그가 섞여 있으면** 신생 블로그 진입 가능성을 메모한다.

### 4. 구글 vs 네이버 콘텐츠 구분

| 플랫폼 | 이 블로그 우선순위 | 이유 |
|---|---|---|
| Google (GitHub Pages) | 높음 / 중간 / 낮음 | |
| Naver | 높음 / 중간 / 낮음 | |

이 블로그는 **GitHub Pages → Google 검색**이 주 채널이다. 네이버는 참고·교차 검증용으로 본다.

### 5. 판단·보류

- 글 작성 가능성: 높음 / 중간 / 낮음 / 보류
- 보류 사유: 예) 네이버는 대형 매체만, 구글은 기회 있음

### 6. 파일 저장

- 경로: `raw-data/naver-related-search/YYYY-MM-DD.md`

### 7. keywords.csv 반영

- `source_hint`에 "Naver 연관검색어 YYYY-MM-DD" 추가
- `naver_related_found` (확장 열 적용 시): Y / N
- `memo`에 구글·네이버 판단 차이 기록
