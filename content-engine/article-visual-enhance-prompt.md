# 시각 요소 2차 패스 프롬프트

**검수 통과 후**(또는 내용 확정 직후) 글에 표·박스·**인포그래픽 3장**을 얹을 때 쓴다.  
규칙: `visual-elements.md`

## 사용 시점

- `article-generation-prompt.md`로 뽑은 초안에 시각 요소가 부족할 때
- 3편 품질 보강(사실·구조)이 끝난 뒤, 4편 레이어를 추가할 때

**순서 SSOT:** `PIPELINE.md` §4 → §5

내용 확정(초안+AI리뷰) → 이 프롬프트(시각+PNG) → Claim Log → checklist → 발행

---

## 글당 이미지 3장 (고정)

| # | 유형 | 역할 | 파일명 예 |
|---|---|---|---|
| 1 | **흐름도** | 절차·분기 (평가 순서 등) | `{slug}-flow.png` |
| 2 | **비교·대조** | 표 1개를 시각 요약 (A vs B) | `{slug}-compare.png` |
| 3 | **상황·유형** | 3가지 케이스·유형별 갈림 | `{slug}-scenarios.png` |

- 저장: `static/images/`
- 본문 삽입: `![alt 텍스트](/images/파일명.png)` + 이탤릭 캡션 1줄 (기준일)
- Mermaid는 **초안·검수용** — 발행 전 **1번 흐름도 PNG로 교체** (또는 Mermaid 없이 PNG만)
- 스타일: 흰 배경, 그린 포인트 `#166534`, 플랫 인포그래픽, **스크린샷·실사 금지**

## 프롬프트 (복사용)

~~~~
아래 글에 시각 요소를 추가해줘. 본문 **사실관계·법령 표현은 바꾸지 마**.

대상 파일: content/posts/{target_slug}.md
참고: content-engine/visual-elements.md

## 할 일

### 마크다운 (기존)

1. **핵심 요약 박스** — 서론 직후 없으면 추가 (인용 블록, 3줄 이내)
2. **주의 박스** — 본문 중간 흔한 함정 1곳 (없으면 추가)
3. **비교 표** — memo·content_type에 맞는 표 1개 (글당 **2개 이내**)

### PNG 인포그래픽 3장 (필수)

4. **흐름도** — 절차·분기가 헷갈리는 구간 → `static/images/{target_slug}-flow.png`
5. **비교·대조** — 핵심 비교표 옆 시각 요약 → `static/images/{target_slug}-compare.png`
6. **상황·유형** — 글의 3가지 케이스·유형 → `static/images/{target_slug}-scenarios.png`

각 이미지마다:
- 본문 적절한 위치에 `![alt](/images/....png)` 삽입
- alt: 그림을 한 문장으로 (키워드 1회 자연스럽게)
- 바로 아래 이탤릭 캡션 1줄 (`*도식: … (YYYY년 M월 기준).*`)

이미지 생성은 Cursor 이미지 생성 도구로 **플랫 벡터 인포그래픽**으로 만든다. 한글 라벨은 글 내용과 맞추고, 생성 후 **눈으로 오타 검수**.

## 하지 말 것

- 세율·기간·비용 **단정** 추가
- 국세청·앱 **캡처 이미지**
- 구글 이미지·타사 감정평가서 **무단 사용**
- 「작성자가 몰라서」 등 자기 고백 문장
- `draft: true` 변경

## 출력

- `static/images/`에 PNG 3개 저장
- `content/posts/{target_slug}.md` 덮어쓰기
- 변경 요약 5줄 이내 (표·박스·이미지 3장 위치)
